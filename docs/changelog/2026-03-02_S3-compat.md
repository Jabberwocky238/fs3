# S3 Compatibility Changelog

## 2026-03-02 — ETag Handling (S3/MinIO Compatible)

### Fixed

- **Multipart Upload ETag**: Computed as `MD5(MD5(part1) || MD5(part2) || ... || MD5(partN))-N`, matching the S3/MinIO specification. Previously returned an empty string.
- **Copy Object ETag**: Now correctly copies the ETag from the source object. Previously returned an empty string.
- **Copy Part ETag**: Now computes MD5 of the copied data for each part. Previously returned an empty string.
- **ETag Quoting**: All ETag values in HTTP headers and XML responses are now wrapped in double quotes (`"etag_value"`), matching AWS S3 and MinIO behavior.

### How It Works

Single-part upload:
```
ETag = MD5(content)
```

Multipart upload:
```
e1 = MD5(part-1)
e2 = MD5(part-2)
...
eN = MD5(part-N)

ETag = MD5(e1 || e2 || ... || eN)-N
```

Example: `"ceb8853ddc5086cc4ab9e149f8f09c88-5"`
